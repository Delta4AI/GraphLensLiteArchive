\section{Program description and methods}\label{materials_and_methods}
Graph Lens Lite (GLL) is a free, open-source web application for visualization and exploration of networks. It is written using HTML, CSS and JavaScript and runs on any modern browser without requiring additional installations. GLL uses G6 \citep{wang_g6_2021} (MIT License) for graph rendering, and SheetJS \citep{clapton_sheetjs_nodate} (Apache License 2.0) for interacting with spreadsheet data. The source code for GLL, along with templates and documentation, is available at \texttt{https://github.com/Delta4AI/GraphLensLite}.

\subsection{Graph Filtering}\label{materials_and_methods_graph_filtering}
Graph Lens Lite provides graph filtering capabilities through easy-to-use UI elements or an AST-based query editor for complex filtering operations. The query editor uses a left-to-right syntax with support for boolean operators (AND, OR, NOT) and nested queries using parentheses. Help is available via an adjacent icon to the query interface, as shown in figure \ref{fig:query_editor_with_help}.

\subsection{Data Editor}\label{materials_and_methods_data_editor}
Graph Lens Lite includes an integrated data editor to explore and modify the original data without external tools. It supports column sorting, cell editing, addition and deletion of nodes and edges, export of the tabular data as an Excel file, and resetting the table to its original state. The data editor interface is shown in figure \ref{fig:data_editor}.

\subsection{Network Metrics}\label{materials_and_methods_network_metrics}
Graph Lens Lite computes network metrics to identify key nodes within the graph structure. Users can select a metric to choose one or more nodes from a ranked list for further analysis. Next to the node selection panel, graph-level statistics are displayed. Each method includes in-app help describing the methodology and references to relevant publications, as shown in figure \ref{fig:network_metrics_help_degree_centrality}.

\subsection{Comparison to other tools}\label{materials_and_methods_comparison_to_other_tools}

Some tools (OmicsNet 2.0 \cite{zhou_omicsnet_2022}, STRING \cite{szklarczyk_string_2025} and NAViGaTOR \cite{brown_navigator_2009}) prioritize data integration, enrichment analysis, and access to extensive biological databases over visualization and customization, integrating the network viewer as a helper and not as the focus of the application. Cytoscape \cite{shannon_cytoscape_2003} and Gephi \cite{bastian_gephi_2009} are considered gold standards for network visualization with an exhaustive plugin ecosystem and rich layouting capabilities, but are desktop-focused and lack portability or modern web-app features. PINA 3.0 \cite{du_pina_2021} integrates the PINA database that is built from a set of publicly available data sources and focuses on network analysis, target identification and clustering in the oncological space, but provides no offline installation methods, depends on an API and does not provide their source code. BioLinker \cite{dang_biolinker_2017} integrates a database derived from text-mined PPI interactions from publications gathered via PMC, using Odin/REACH \cite{valenzuela-escarcega_large-scale_2018}. The source code of BioLinker is available, but the API it depends on is not reachable and the project seems to be not maintained, making it impossible to run the application. Two competitors come close to the functionality of Graph Lens Lite: Gephi Lite \cite{noauthor_gephigephi-lite_2025} is a lightweight and efficient competitor to Graph Lens Lite, using sigma.js \cite{jacomy_jacomyalsigmajs_2025} for efficient WebGL-based rendering, however compromises on customization, filtering, or advanced styling. Graphia \cite{freeman_graphia_2022} shows promises of a powerful 2D/3D rendering engine, providing both a desktop and web client. Similar to Gephi Lite, Graphia puts less focus on user-friendly styling, highlighting, grouping and filtering, and more on efficiency, enrichment and layouting. 

Graph Lens Lite is a particularly portable application with a focus on visual customization, compared to other tools. The entire source code and an average sized graph containing approximately 1000 nodes can be packed into an archive of less than 1MB size. While other tools may require the installation of dependencies (R, JDK, databases, ..), complex installation routines and the execution of potentially unsafe scripts containing binary data, GLL solely relies on a web-browser. The minimum package contains three JavaScript files, one HTML file and one spreadsheet data file. In addition, GLL does not query any external APIs that might deprecate over time and would require an active internet connection, making the package self-sustainable and long-lasting.
